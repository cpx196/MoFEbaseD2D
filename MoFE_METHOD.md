# MoFE for GPT-2 Small

## Scope

This implementation converts only GPT-2 Small blocks 9, 10, and 11. Attention,
normalization, embeddings, the first nine MLPs, and the tied language-model head
remain unchanged.

Each converted MLP contains one always-on dense shared expert and 16 factorized
private experts. A token-choice router selects exactly three private experts per
token. The shared expert is excluded from routing and is not multiplied by a
router probability.

```text
hidden state x -> shared dense MLP --------------------+
               -> router -> private top-3 -> weighted + -> output
```

The output is

```text
y = shared_mlp(x) + private_scale * sum(g_e(x) * private_e(x))
```

where the selected private weights sum to one. `private_scale` is one for the
defined MoFE architecture. The training script can warm it from zero to one to
avoid injecting the randomly initialized private branch at the first step.

## Factorized private experts

GPT-2 Small uses hidden size `d=768` and MLP size `f=3072`. With rank `r=576`,
expert `e=4i+j` is generated by:

```text
W1_e = A1_i C1_e B1_j
W2_e = A2_i C2_e B2_j
```

The four A banks and four B banks are shared through a 4x4 Cartesian product.
Each of the 16 experts owns two independent core matrices and two private bias
vectors.

| Parameter | Shape |
| --- | --- |
| `A1` | `[4, 3072, 576]` |
| `B1` | `[4, 576, 768]` |
| `C1` | `[16, 576, 576]` |
| `A2` | `[4, 768, 576]` |
| `B2` | `[4, 576, 3072]` |
| `C2` | `[16, 576, 576]` |
| private bias 1 | `[16, 3072]` |
| private bias 2 | `[16, 768]` |
| router | `768 -> 16` |

Private execution applies the factors in the order `x -> B -> C -> A`; it does
not materialize a complete expert matrix per token. The private path uses
GPT-2's activation and applies dropout once after its second projection.

## Initialization

The shared expert is a deep copy of the original Hugging Face GPT-2 MLP. Since
GPT-2 `Conv1D` stores weights as `[in, out]`, source weights are transposed to
linear notation before factor initialization:

```text
W1 = c_fc.weight.T
W2 = c_proj.weight.T
```

All four copies of each A/B bank receive the same row or column slice:

```text
A1_i = W1[:, :576]       B1_i = W1[:576, :]
A2_i = W2[:, :576]       B2_i = W2[:576, :]
```

Core matrices are independent `Normal(0, 0.025^2)` samples. Private biases are
zero. Router weights use `Normal(0, 0.02^2)` and router bias is zero. This is a
row/column slice initialization, not an exact Nystrom reconstruction. Therefore
the full MoFE output does not initially equal the dense output; only the
shared-only output does.

## Routing and losses

For router logits `z`, each token selects top-3 experts and normalizes only the
three selected logits. There is no capacity limit and no token dropping.

Training uses:

```text
L = L_LM + 0.01 * L_balance + 0.001 * L_z
```

`L_balance` uses full router probabilities and top-k assignment frequencies.
`L_z` is the mean squared `logsumexp` of router logits. Runtime statistics
include per-expert counts, assignment fractions, load CV, max/min load ratio,
router entropy, unused expert count, and the raw private/shared output norm
ratio before application of the warmup scale.

## Main configuration

The checked-in configuration is
[mofe_gpt2_last3_e16_k3.json](MoFE/configs/mofe_gpt2_last3_e16_k3.json).

| Setting | Value |
| --- | --- |
| Base checkpoint | `openai-community/gpt2` |
| Converted blocks | `9, 10, 11` |
| Shared experts | one per converted block, always active |
| Private experts | 16 per converted block |
| Routing | token-choice top-3 |
| Rank | 576 (`0.75 * 768`) |
| Core standard deviation | 0.025 |
| Precision | bf16 in the training entry point |
| Initial training plan | 200 optimizer steps |
| Distributed mode | Accelerate/DDP, no expert parallelism |

The verified GPT-2 Small total is `209,595,696` parameters with Transformers
5.13.1. Of these, the three MoFE layers contain 14,167,296 shared-expert,
53,084,160 factor-bank, 31,850,496 core, 184,320 private-bias, and 36,912
router parameters.

## Validation

The local unit suite uses a tiny randomly initialized GPT-2 and does not require
a dataset or network access:

```bash
python -m unittest MoFE.tests.test_mofe
```

Validate the real GPT-2 initialization with:

```bash
python -m MoFE.validate_initialization \
  --output ./data/pxchen/results/mofe_gpt2_last3_e16_k3/initialization_report.json
```

The report checks shared weights, dense A/B slices, dense versus shared-only
logits, dense versus full-MoFE logits, parameter shapes, norms, and trainability.

## 200-step training entry point

No dataset is selected in code or configuration. Training requires either a
Hugging Face dataset name or a local text/JSON file at launch time:

```bash
accelerate launch --num_processes 4 -m MoFE.train \
  --dataset-name DATASET_TO_BE_SELECTED \
  --text-column text \
  --max-steps 200
```

or:

```bash
accelerate launch --num_processes 4 -m MoFE.train \
  --train-file /path/to/training.jsonl \
  --text-column text \
  --max-steps 200
```

Defaults are one sequence per device, eight gradient-accumulation steps, bf16,
20 optimizer warmup steps, and a 200-step private-branch warmup. The longer
private ramp is deliberate: in a real GPT-2 initialization smoke test, the
shared-only maximum logit difference was exactly zero, while immediately
enabling the random private branch produced a maximum logit difference of
126.83 for the test prompt. Dataset choice, sequence packing suitability,
effective batch size, and learning rate must be confirmed before the run.

Checkpoints contain the MoFE configuration and Accelerate training state. The
final export also contains a standalone state dictionary loadable through
`MoFE.load_mofe_checkpoint`.
